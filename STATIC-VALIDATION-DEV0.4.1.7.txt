OBJECTIFS DU BUILD
- Le type de service Facilities actif départage les modes SIA publiés sur un même canal.
- La fréquence active détermine l'organisme ; le nom prononcé est facultatif.
- Un appel explicite vers une autre station ou un autre service reste sans réponse.
- L'historique radio est séparé par organisme et survit aux changements normaux de fréquence, d'ATIS et d'aérodrome.
- Une nouvelle session de vol efface contacts, salutations, indicatifs abrégés, ATIS, roulage et collationnements, y compris avec le même avion et le même indicatif après Restart Flight.
- Les opérations sol ne sont jamais délivrées par un SIV/FIS régional.
- Aucun aérodrome ni fréquence de validation n'est codé en dur dans le code de production.

COUVERTURE DES NOUVELLES RÉGRESSIONS
- Tour Facilities prioritaire sur A/A lorsque le canal SIA est partagé.
- Sécurité silencieuse conservée en l'absence de signal Facilities fiable.
- Premier contact avec indicatif complet et autorisation ultérieure de l'indicatif abrégé par organisme.
- Première transmission contenant déjà la demande de roulage traitée sans demander de répéter.
- « bonjour » non répété sur le même organisme.
- nouveau premier contact lors du passage Sol vers Tour.
- « rebonjour » / « de retour » après ATIS avec conservation du contact Tour.
- changement de contexte aérodrome sans destruction de l'historique de vol.
- ResetFlightSession effaçant tout l'historique radio et l'état sol.
- appel clair vers un autre organisme produisant le silence.
- SIV/FIS régional refusant roulage, alignement et décollage.
- ATIS, message enregistré et auto-information sans dialogue.

VALIDATION CI AVANT COLLECTE SIA
Le workflow build-windows.yml exécute dans cet ordre :
1. contrôle Git, CRLF et SHA-256 ;
2. restore des projets ;
3. compilation PHONIE avec avertissements traités comme erreurs ;
4. compilation des projets de tests avec avertissements traités comme erreurs ;
5. Smoke Tests ;
6. Core Tests ;
7. prépublication autonome Windows x64 avec contrôle de PHONIE.exe ;
8. seulement ensuite, génération et validation de la base SIA ;
9. publication finale Windows x64.

LIMITATION DE L'ENVIRONNEMENT DE PRÉPARATION
Le SDK .NET/Windows n'est pas disponible dans cet environnement. La compilation WPF et l'exécution des tests C# ne peuvent donc pas être lancées localement ici. Le ZIP est préparé pour que GitHub Actions effectue ces contrôles exécutables avant toute opération réseau longue. Aucun résultat de compilation de DEV0.4.1.7 n'est revendiqué avant ce passage CI.
