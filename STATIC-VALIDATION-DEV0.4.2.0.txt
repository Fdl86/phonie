COUVERTURE DES NOUVELLES RÉGRESSIONS
- Le nom de station est inutile après établissement du contact.
- « pour un départ », « avec information », « en approche » et « au sol » ne deviennent pas des stations.
- Une station active déformée peut être rapprochée sans faux silence.
- Une autre Tour connue reçoit l'identifiant de la station active et une fréquence sûre, ou une demande de vérification.
- Une demande de roulage vers le point d'attente ne devient pas une déclaration de disponibilité au départ.
- Une demande intersection survit à certaines déformations du mot « prêt » tout en restant contrôlée par la position réelle.
- « retour avec vous » est reconnu sans exiger « de ».
- Les protections ATIS, message enregistré, A/A et SIV/FIS régional sont conservées.
- La remise à zéro Restart Flight/menu/nouvelle session et l'historique par organisme sont conservés.

VALIDATION CI AVANT PUBLICATION
Le workflow build-windows.yml exécute :
1. précontrôle source, attributs Git et SHA-256 ;
2. restore .NET ;
3. compilation PHONIE avec avertissements traités comme erreurs ;
4. compilation des projets de tests avec avertissements traités comme erreurs ;
5. Smoke Tests ;
6. Core Tests ;
7. prépublication autonome Windows x64 et présence de PHONIE.exe ;
8. tests, génération et validation SIA ;
9. publication de l'artefact portable.

LIMITATION DE L'ENVIRONNEMENT DE PRÉPARATION
Le SDK .NET/Windows n'est pas installé dans cet environnement. La compilation WPF, les Smoke Tests et les Core Tests ne sont donc pas déclarés réussis localement. Le workflow GitHub Actions est configuré pour arrêter la chaîne avant génération ou publication au premier avertissement, échec de compilation ou test en erreur.
